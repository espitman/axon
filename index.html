Private LAN bridge for your own devices

Mirror Android SMS, calls, and media controls without a cloud relay.

Axon connects a Sender phone and a Receiver phone over your local network. The Receiver shows mirrored alerts, keeps a local inbox, and exposes notifications and media controls to a connected watch through Android's normal system pipeline.

What it does

One bridge, two roles.

SMS mirror and inbox

Sender captures SMS events directly and resolves contact names when allowed. Receiver stores messages in a local thread-based inbox with unread badges and Persian-friendly text rendering.

Call mirroring

Incoming and ongoing call state is mirrored to Receiver. Receiver can show a full-screen call alert, keep a call log, and send an experimental reject command back to Sender.

Media and watch controls

Sender streams active media metadata and artwork. Receiver creates a shadow media session so phone, notification, and watch controls can route commands back to Sender.

Architecture

Local by design.

Axon uses Ktor WebSockets over LAN. Receiver hosts the bridge on port 8080; Sender discovers or connects to it and sends structured JSON events. There is no Axon server.

Setup

Install on both Android phones.

1

Start Receiver

Install Axon on the phone that should show alerts and connect to your watch. Select Receiver mode and start the bridge.

2

Start Sender

Install Axon on the phone that receives the real SMS, calls, and media playback. Select Sender mode.

3

Connect locally

Use network scan or manual IP entry. Both phones must be on the same trusted LAN or hotspot network.

4

Allow watch alerts

In your watch companion app, allow notifications from Axon. Watch controls use Android's normal notification and media-session paths.

Security and limits

Use Axon on networks you trust.

The current release intentionally prioritizes local utility over security. It does not yet implement pairing, encryption, identity verification, or revocation. Use it only on trusted private networks.

Known limitations

  • Full-screen call alerts depend on Android and OEM settings.
  • Reject call is experimental and may return false on some devices.
  • Watch behavior depends on the phone companion app and its allowlist.
  • Local archive data stays on the Receiver; it is not a system SMS database.

Troubleshooting

Fast checks when something does not mirror.

Receiver gets SMS but watch does not

The Axon bridge is working. Open the watch companion app and allow notifications from Axon, then check battery restrictions for the companion app.

Network scan finds nothing

Confirm both devices are on the same subnet, disable VPN/proxy, and try manual IP. Sender also retries discovery after Wi-Fi changes while the bridge is active.

Watch media buttons do nothing

Check Receiver diagnostics for the shadow media session and silent playback anchor. Axon uses these so watch controls route into the Receiver media session.

Full-screen call does not appear

On Receiver, allow full-screen call alerts in Android settings and keep the Incoming Calls notification channel at high priority.
