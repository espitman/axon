Private bridge for your own Android devices

Mirror Android SMS, calls, and media controls over local Wi-Fi or your own ntfy relay.

Axon connects a Sender phone and a Receiver phone. The Receiver shows mirrored alerts, keeps a local inbox and call log, and exposes notifications and media controls to a connected watch through Android's normal system pipeline.

Current status: Axon is experimental. LAN mode is for trusted local networks. ntfy mode encrypts Axon payloads with a shared pair secret, but your relay still sees metadata such as topic names, timing, message sizes, and account activity. A personal ntfy server can run without ntfy authentication; authentication and deny-by-default ACLs are recommended for shared or public relay servers.

What it does

One app, two roles.

SMS mirror and inbox

Sender captures SMS events directly and resolves contact names when allowed. Receiver stores messages in a local thread-based inbox with unread badges, conversation view, and long-press deletion.

Call mirroring

Incoming and ongoing call state is mirrored to Receiver. Receiver can show a full-screen call alert, keep a call log, delete call entries, and send an experimental reject command back to Sender.

Media and watch controls

Sender streams active media metadata. Receiver creates a shadow media session so phone, notification, and watch controls can route commands back to Sender.

Transport

Choose direct LAN or self-hosted relay.

LAN mode uses a direct Ktor WebSocket bridge. ntfy mode uses two topics on a self-hosted ntfy server and encrypts Axon payloads in the app before publishing. ntfy username/password is optional for personal servers.

Setup

Install on both Android phones.

1

Start Receiver

Install Axon on the phone that should show alerts and connect to your watch. Select Receiver mode and start the bridge.

2

Start Sender

Install Axon on the phone that receives the real SMS, calls, and media playback. Select Sender mode.

3

Choose transport

Use LAN scan/manual IP on the same network, or configure a self-hosted ntfy server with matching pair ID and pair secret.

4

Allow watch alerts

In your watch companion app, allow notifications from Axon. Watch controls use Android's normal notification and media-session paths.

Security and limits

Useful, private, still experimental.

ntfy relay payloads are encrypted with the Axon pair secret, but this is not the same as a hardened public messaging product. Private pair secrets, unguessable topics, optional server ACLs, and careful Android permission setup still matter.

Known limitations

  • ntfy server metadata remains visible to the relay operator.
  • Full-screen call alerts depend on Android and OEM settings.
  • Reject call is experimental and may return false on some devices.
  • Watch behavior depends on the phone companion app and its allowlist.
  • Large media artwork is compacted over ntfy and omitted only if it still exceeds the safe payload limit.
  • Local archive data stays on the Receiver; it is not a system SMS database.

Troubleshooting

Fast checks when something does not mirror.

ntfy auth setup

For a personal open relay, leave both username and password/token empty. If your relay requires auth, fill both fields and verify each user has the correct read/write ACL.

Wrong pair ID or secret

Both phones must share the same pair ID and pair secret. Wrong pair IDs are ignored; wrong pair secrets fail payload authentication and are ignored.

Server unreachable

Test the base URL with a browser or curl, then check DNS, HTTPS, reverse proxy, firewall, and Android network restrictions. Use reconnect after changing settings.

Commands do not reach Sender

Confirm Receiver can publish to axon-<pairId>-to-sender and Sender can subscribe to it. For media, Sender also needs Notification Access to see the active session.

Payload too large

ntfy mode keeps messages small. Large artwork is compacted first and omitted only if it still exceeds the safe payload limit, while media metadata and controls should continue working.

Receiver gets SMS but watch does not

The Axon bridge is working. Open the watch companion app and allow notifications from Axon, then check battery restrictions for the companion app.
